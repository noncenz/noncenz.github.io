## Become Fred

### Intended Path

Some quick enumeration as Barney leads us to that `/usr/share/abc` path that was leaked by the webserver. Here we find the `certs` folder with some additional certificate files, including certs for Fred. Unfortunately we can‚Äôt read them directly. We do, however, have `sudo` power for the `certutil` application - so perhaps we can simply make our own certificates for Fred. 

The `certutil` application doesn‚Äôt seem to take the command line parameters we expect. 

```terminal
certutil -h
                                                                                                                     
Cert Tool Usage:                                                                                                     
----------------                                                                                                     
                                                                                                                     
Show current certs:                                                                                                  
  certutil ls

  certutil [username] [fullname]
```

Investigating further we find that it isn‚Äôt the standard package but rather it eventually runs the `certs.js` program in `/usr/share/abc/dist`. 

We run it as:

`sudo /usr/bin/certutil fred "Fred Flintstone" > /tmp/fred.txt`
